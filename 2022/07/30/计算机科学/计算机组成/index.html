    计算机系统概述

    计算机硬件发展

    计算机的四代变化

    • 第一代计算机：电子管时代，主存用延迟线或者磁鼓存储信息，机器语言编程
    • 第二代计算器：晶体管时代，开始出现高级语言及其编译程序，磁芯存储器，有了操作系统的雏形
    • 第三代计算机：中小规模集成电路时代，半导体存储器，开始有了分时操作系统
    • 第四代计算机：超大规模集成电路时代，产生微处理器

    计算机系统层次结构

    • 硬件系统和软件系统构成了一个完整的计算机系统

    计算机硬件

    冯诺依曼机基本思想

    • 采用“存储程序”的工作方式（程序和数据送入主存后才能执行）
    • 计算机硬件系统由运算器、存储器、控制器、输入设备和输出设备五大部件组成
    • 指令和数据以同等地位存储在存储器中，形式上没有区别，但计算机应能区分它们。
    • 指令和数据均用二进制代码表示。指令由操作码和地址码组成，操作码指出操作的类

    计算机的功能部件

    主存储器逻辑图

    • 输入设备
    • 输出设备
    • 存储器
      • 主存储器（内存储器），CPU 能直接访问；按地址存取方式
        • 存储体存放二进制信息 > 包含若干存储单元 > 包含若干存储元件 > 存储一位二进制代码 0 或 1
        • 地址寄存器 MAR 存放访存地址，经过地址译码后找到所选的存储单元（位于 CPU）
        • 数据寄存器 MDR 用于残存要从存储器中读或写的信息（位于 CPU）
        • 时序控制逻辑用于产生存储器所需的各种时序信号
      • 辅助存储器（外存储器），先调入主存，才能被 CPU 访问
    • 运算器
      • 核心是算术逻辑单元 Arithmetic and Logical Unit, ALU。
      • 包含若干通用寄存器，用于暂存操作数和中间结果
        • 累加器 ACC、乘商寄存器 MQ、操作数寄存器 X、变址寄存器 IX、基址寄存器 BR，前三个寄存器是必备的
        • 内存状态寄存器 PSW，也称标志寄存器，用于存放 ALU 运算得到的一些标志信息或处理机的状态信息（溢出）
    • 控制器
      • 是计算机的指挥中心，指挥各部件自动协调进行工作
      • 程序计数器 PC、指令寄存器 IR、控制单元 CU 组成
        • PC 用来存放将执行指令地址，可自动加一形成下一条指令，与 MAR 有直接通路
        • IR 用来存放当前指令，其内容来自主存的 MDR
          • OP(IR) > CU 分析指令并发出各种微操作命令序列
          • AD(IR) > MAR 取数
    • CPU = 运算器 + 控制器；主机 = CPU + 存储器； 除主机外其它硬件统称为外部设备（外设）

    冯诺依曼结构的模型机

    • CPU 和主存之间通过一组总线相连，总线中有地址、控制和数据三组信号线
      • MAR 地址信息送到地址线，用于指向读/写操作的主存存储单元
      • 控制线中有读/写信号线，指出数据是从 CPU 写入主存还是从主存读出到 CPU
      • 根据读/写操作将 MDR 数据送到数据线/数据线上的数据接收到 MDR 中

    计算机软件

    • 系统软件

      • 保证计算机系统高效、正确运行的基础软件
      • 操作系统 OS、数据库管理系统 DBMS、语言处理程序、分布式软件系统、网络软件系统、标准库程序、服务性程序等

    • 应用软件

      • 用户为解决某个应用领域中的各类问题而编制的程序

    • 三个级别的语言

      • 机器语言，又称二进制代码语言，是计算机唯一可以直接识别和执行的语言
      • 汇编语言：通过汇编程序转换为机器语言程序，才能在计算机的硬件系统上执行
      • 高级语言：编译程序（> 汇编语言程序）> 机器语言程序

    • 翻译程序

      • 汇编程序：将汇编语言翻译成机器语言程序
      • 解释程序：将源程序中的语句按执行顺序逐条翻译成机器指令并立即执行
      • 编译程序：将高级语言程序翻译成汇编语言或机器语言程序

    计算机系统层次结构

    计算机系统的多级层次结构

    • 第一级是微程序机器层，是硬件层，由机器硬件直接执行微指令
    • 第二级是传统机器语言层，是机器层，由微程序解释解释机器指令
    • 第三级是操作系统层，又称混合层，由操作系统程序实现，操作系统程序由机器指令和广义指令组成
      • 广义指令是为了扩展机器功能设置的，是由操作系统定义和解释的软件指令
    • 第四级是汇编语言层，为用户提供一种符号化语言，由汇编程序支持和执行
    • 第五级是高级语言层，面向用户，方便用户编写应用程序，由各种高级语言编译程序支持和执行
    • 还可以有应用程序层，由解决实际问题和应用问题的处理程序组成
    • 没有配备软件的纯硬件系统成为裸机
    • 第三层到第五层称为虚拟机

    计算机系统的工作原理

    • ”存储程序“工作方式
      • 程序执行前，需将程序所含指令和数据送入主存，程序启动后无需人员干预，自动逐条完成指令的取出和执行任务
      • 每条指令的执行过程包括：从主存取指令、对指令进行译码、计算下一条指令地址、起操作数并执行、将结果送回存储器

    • 从源程序到可执行文件
      • 预处理阶段：预处理器 cpp 对源程序中以字符 # 开头的命令进行处理
      • 编译阶段：编译器 ccl 对预处理后的源程序进行编译
      • 汇编阶段：汇编器将其翻译成机器语言指令
      • 链接阶段：链接器 ld 将多个可重定向目标文件和标准库函数合并为一个可执行(目标)文件，

    程序执行过程描述

    • 取数指令（送至 ACC）执行过程描述
      • 取指令 PC > MAR > M > MDR > IR
      • 分析指令 OP(IR) > CU，操作码
      • 执行指令 Ad(IR) > MAR > M > MDR > ACC，地址码
      • 取完指令 (PC) + 1 > PC

    计算机性能指标

    • 计算机的主要性能指标

      • 字长：一次整数运算所能处理的二进制数据位数，一般等于内部寄存器的大小（内部数据总线宽度）
      • 数据通路带宽：外部数据总线的宽度
      • 主存容量：MAR 位数表反映存储单元个数，反映可寻址范围的最大值（不一定是存储器的存储容量）
      • 运算速度
        • 吞吐量：主要取决于主存的存取周期
        • 响应时间：CPU 时间 + 等待时间
        • CPU 时钟周期：节拍脉冲或 T 周期，即主频的倒数，是 CPU 中最小的时间单位
        • 主频（CPU 时钟频率）：主频越高，执行指令的速度越快
        • CPI，Clock cycle Per Instruction：执行一条指令所需要的时钟周期，是一个平均值
        • CPU 执行时间：指运行一个程序所花费的时间 = CPU 时钟周期/主频 = 指令条数xCPI/主频（CPI 和指令条数相互制约）
        • MIPS：每秒执行多少百万条指令（不同机器有指令差异）
        • XFLOPS：每秒执行的浮点数操作
      • 基准程序
        • 专门用来今次那个性能评价的一组程序

    • 几个专业术语

      • 系列机
      • 兼容
      • 软件可移植性
      • 固件：将程序固话在 ROM 中组成的部件。目前操作系统已实现了部分固化

    定点数的编码表示

    • 原码：用机器数最高位表示数的符号，其余各位表示数的绝对值

    • 补码：正数同原码；负数符号位取 1，其余位取反并 +1（真补互转）

    • 反码：正数同原码；负数符号位取 1，其余位取反

    • 移码：在真值 X 上加上一个常数（偏置值）

    • 补码表示整数的优势

      • 0 唯一
      • 运算规则简单，符号位可参与运算
      • 多表示一个最小负数
